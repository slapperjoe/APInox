What Is Rust, and Why Should You Care?

⏱ ~20 min 📋 Conceptual — no code files needed 🎯 Language overview, Cargo, interview prep

1. What Is Rust?

Rust is a systems programming language created at Mozilla Research, released in 2015. It's now governed by the independent Rust Foundation (backed by AWS, Google, Microsoft, Meta).

It exists to answer one question:

"Can we have the performance of C/C++ without the memory bugs?"
The answer: yes — by making memory safety a compile-time guarantee, not a runtime check.

✅ True or False
Rust uses a garbage collector to manage memory.
False! Rust has no garbage collector. Memory is managed by the ownership system — the compiler proves at compile time that memory is freed at the right moment. No runtime cost, no GC pauses.

2. The Three Core Promises

① Performance

No GC means no GC pauses. Zero-cost abstractions — high-level code compiles down to the same machine code as hand-written C. Rust uses the same LLVM backend as Clang (C/C++), so it gets decades of optimisation for free.

② Memory Safety

No null pointer dereferences. No dangling references. No buffer overflows. No data races. None of these are checked at runtime — the compiler rejects unsafe code.

③ Fearless Concurrency

Thread-safety is enforced at compile time via the Send and Sync traits. If your multithreaded code compiles, it is guaranteed to be free of data races.

🔘 Multiple Choice
What guarantees Rust's memory safety?
✓ The borrow checker! It's a compile-time pass that enforces ownership rules. If your code compiles, the memory is safe. There's no runtime overhead — no GC, no ref-count updates, nothing.

5. Project Structure

my-project/
├── Cargo.toml          ← package.json + .csproj (metadata, dependencies)
├── Cargo.lock          ← package-lock.json (exact resolved versions)
├── src/
│   ├── main.rs         ← entry point for a binary
│   ├── lib.rs          ← entry point for a library
│   └── some_module.rs  ← additional modules
└── tests/
    └── integration.rs  ← integration tests (outside src/)

Dependencies live in Cargo.toml and are downloaded from crates.io (the npm registry of Rust):

[dependencies]
serde = { version = "1.0", features = ["derive"] }
reqwest = "0.12"
tokio = { version = "1.45", features = ["full"] }
features = [...] — Feature flags let you opt into optional functionality of a crate. Without features = ["derive"] on serde, the #[derive(Serialize)] macro simply doesn't exist. You only compile what you need.

6. The Compilation Pipeline

Your .rs files
     ↓
  rustc (Rust compiler)
     ↓  type checking + borrow checking
     ↓  macro expansion
     ↓  MIR (Mid-level IR)
     ↓
  LLVM backend
     ↓  full optimisation passes
     ↓
  Native binary (.exe / ELF / .dylib)

The binary is self-contained — no runtime needed (unlike .NET which requires the CLR, or Node.js apps which need the runtime). Debug builds compile fast but run slower. Release builds (--release) compile slowly but run at full speed.

Key Interview Questions

Q: What is ownership? ▸

Every value has exactly one owner. When the owner goes out of scope, the value is dropped (freed). This eliminates the need for a GC.

Q: What is the borrow checker? ▸

A compile-time analysis that enforces: at any given time, you can have either one mutable reference OR any number of immutable references — never both. This prevents data races and use-after-free.

Q: What is a trait? ▸

An interface definition. Types implement traits. Unlike C# interfaces, you can implement a trait for a type you didn't define (called an "orphan rule" exception).

Q: What is monomorphisation? ▸

When a generic function fn foo<T>() is called with multiple concrete types, Rust generates separate compiled functions for each. Faster than virtual dispatch (no vtable lookup), but can increase binary size.

Q: String vs &str? ▸

String is owned, heap-allocated, growable. &str is a borrowed string slice — a view into some string data (often a literal or a slice of a String). You can't grow a &str.

✅ True or False
Rust can compile to WebAssembly (WASM) and run in a browser.
True! Rust is a first-class WASM target. Tools like wasm-pack and wasm-bindgen let you compile Rust to WASM and call it from JavaScript. It's one of the most popular use cases for Rust.
