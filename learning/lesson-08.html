Pattern Matching Deep Dive

⏱ ~30 min 📁 testing/assertion_runner.rs, workflow/engine.rs

0 of 5 questions answered

Pattern matching is one of Rust's most powerful features — and one of the most enjoyable. C# has basic switch expressions, but Rust's match goes much further. Once it clicks, you'll miss it in every other language.

1. match — Exhaustive Switch on Steroids

From testing/assertion_runner.rs — matching an enum with data in each variant:

match assertion {
    // Destructure struct variants right in the pattern:
    AssertionType::XPath { xpath, expected } => {
        run_xpath_assertion(response, &xpath, &expected)
    }
    AssertionType::JsonPath { path, expected } => {
        run_jsonpath_assertion(response, &path, &expected)
    }
    AssertionType::Regex { pattern, expected } => {
        run_regex_assertion(response, &pattern, expected.as_deref())
    }
    AssertionType::Contains { value } => {
        run_contains_assertion(response, &value)
    }
    AssertionType::StatusCode { code } => {
        run_status_code_assertion(actual_status, code)
    }
    AssertionType::ResponseTime { max_ms } => {
        run_response_time_assertion(elapsed_ms, max_ms)
    }
}

The compiler refuses to compile if you miss a variant. No fall-through bugs.

// Catch-all patterns:
match value {
    1 => println!("one"),
    2 | 3 => println!("two or three"),   // OR pattern
    4..=9 => println!("four to nine"),   // inclusive range
    _ => println!("something else"),     // wildcard — must be last
}

// Matching a tuple:
match (x, y) {
    (0, 0) => println!("origin"),
    (x, 0) => println!("on x-axis at {x}"),
    (0, y) => println!("on y-axis at {y}"),
    (x, y) => println!("at ({x}, {y})"),
}

2. Guards — Adding if Conditions to Arms

match temperature {
    t if t < 0   => println!("Freezing"),
    t if t < 15  => println!("Cold"),
    t if t < 25  => println!("Comfortable"),
    _            => println!("Hot"),
}

// Guards also work with destructuring:
match point {
    (x, y) if x == y => println!("On the diagonal"),
    (x, y) if x > 0 && y > 0 => println!("First quadrant"),
    _ => println!("Elsewhere"),
}
✍️ Fill in the Blank
Add a guard to this arm so it only matches when max_ms is greater than zero:
AssertionType::ResponseTime { max_ms } => {
    run_response_time_assertion(elapsed, max_ms)
}
if max_ms > 0 — guards are written as if condition after the pattern: AssertionType::ResponseTime { max_ms } if max_ms > 0 => { ... }. If the guard is false, the match continues to the next arm. If no arm matches, the compiler will remind you to add a catch-all.

3. @ Bindings — Name and Test at Once

match status_code {
    // Bind the value to `code` AND test it's in range:
    code @ 200..=299 => println!("Success: {}", code),
    code @ 400..=499 => println!("Client error: {}", code),
    code @ 500..=599 => println!("Server error: {}", code),
    code => println!("Unexpected: {}", code),
}

// Without @, you'd need a guard:
// 200..=299 if true => ... (can't access the value in the arm body)

// Useful with nested patterns too:
match user {
    User { name: n @ "admin", .. } => println!("Admin: {}", n),
    User { name, .. } => println!("User: {}", name),
}
🔘 Multiple Choice
What does the @ binding let you do that a range pattern alone cannot?
✓ Test AND bind! code @ 200..=299 does two things: it checks that the value is in the range, AND binds the matched value to code so you can use it in the arm body. Without @, a range pattern tests the value but gives you no way to refer to it. You'd have to use a guard (c if c >= 200 && c <= 299) and lose the range syntax.

4. if let and while let — Single-Arm Matching

When you only care about one pattern, if let is cleaner than a full match:

// Full match — verbose when you only care about Some:
match config.timeout {
    Some(t) => apply_timeout(t),
    None => {},
}

// if let — much cleaner:
if let Some(t) = config.timeout {
    apply_timeout(t);
}

// Chaining (Rust 1.64+):
if let Some(user) = get_user()
    && let Some(perm) = user.get_permission("admin")
    && perm.is_active() {
    // all three conditions true
}

// while let — loop until pattern fails:
while let Some(item) = queue.pop() {
    process(item);
}

// From workflow/engine.rs pattern:
while let Ok(update) = rx.try_recv() {
    results.push(update);
}
✅ True or False
if let can only be used with Option<T>.
False! if let works with any pattern against any value. Common uses:
if let Some(v) = option { } — Option
if let Ok(v) = result { } — Result
if let MyEnum::Variant(x) = value { } — any enum
if let (a, b, 0) = tuple { } — tuples with specific values
It's essentially a match with one pattern arm and an implicit _ => {}.

5. let-else — The Defensive Pattern

A cleaner alternative to nested if let for early returns:

// Old way — pyramid of doom:
fn process(config: Option<Config>) -> Result<()> {
    if let Some(c) = config {
        if let Some(endpoint) = c.endpoint {
            // now actually do something, 2 levels deep
            send_request(&endpoint)
        } else {
            Err("No endpoint".into())
        }
    } else {
        Err("No config".into())
    }
}

// let-else — stays flat:
fn process(config: Option<Config>) -> Result<()> {
    let Some(c) = config else {
        return Err("No config".into());
    };
    // c is now in scope, no extra nesting
    let Some(endpoint) = c.endpoint else {
        return Err("No endpoint".into());
    };
    send_request(&endpoint)
}
Rule: The else block in let-else must diverge — it must return, break, continue, or panic!. You can't fall through. This is what makes it safe — the variable is guaranteed to be bound if execution continues past the let-else.
🐛 Spot the Bug
This let-else won't compile. Click the problematic line:
fn get_name(user: Option<User>) -> String { let Some(u) = user else { println!("No user found"); }; u.name }
Line 3! The else block must diverge — it cannot fall through. println! just prints and returns (); the block continues executing, which would leave u unbound but not stop the function. Fix it by adding a return: println!("No user"); return String::new(); or just use return "Unknown".to_string();.
🔗 Match It
Match each pattern construct to its best use case:
All matched!
• match → exhaustive enum handling — compiler checks all cases
• if let → "I only care about the Some case" — clean one-liner
• let-else → guard clause / early return — keeps code flat
• while let → loop until pattern fails — perfect for draining channels or queues
